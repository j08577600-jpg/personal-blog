# 文章发布流程 — 从写文章到上线

> V2 版本，仍然基于文件驱动的 MDX 工作流。
> 现在支持两种方式：手工改文件，或使用站内在线写作工作台 V2。

---

## 1. 两种写作路径

### 路径 A：在线写作工作台 V2（推荐）

适合日常新建、编辑、保存、预览、发布。

1. 启动项目并登录：

```bash
cd /home/cjl/projects/personal-blog
npm run dev
```

2. 打开 `http://localhost:3000/dashboard`
3. 使用 GitHub 登录，且账号必须在白名单内
4. 进入“在线写作工作台 V2”
5. 按流程操作：
   - 新建草稿
   - 编辑 frontmatter 与正文
   - 保存
   - 保存并预览
   - 发布 / 取消发布

### 路径 B：手工改文件

适合批量迁移、脚本生成、应急手修。

在 `content/posts/` 下新建或修改：

```text
yyyy-MM-dd_slug.mdx
```

文件名格式：`日期_slug`，日期与 frontmatter 中的 `date` 必须一致。

---

## 2. 在线写作工作台 V2

### 入口

- 控制台：`/dashboard`
- 写作列表：`/dashboard/write`
- 新建草稿：`/dashboard/write/new`
- 编辑文章：`/dashboard/write/[slug]`
- 草稿预览：`/dashboard/preview/[slug]`

### 支持能力

- 新建草稿
- 编辑 `title`、`date`、`slug`、`excerpt`、`tags`、`cover`、`published`
- 编辑正文 Markdown / MDX
- 保存到 `content/posts/*.mdx`
- 保存后预览
- 发布
- 取消发布

### 关键限制

- 仅登录且白名单作者可访问页面与 API
- 只写入 `content/posts/`
- 不支持数据库 / CMS / 多人协作 / 自动 merge
- 不支持未保存内容实时预览，只支持“保存后预览”
- `date` 或 `slug` 变更会导致文件名变化，系统会做冲突检查，不会静默覆盖
- 保存时会检查文件修改时间；若他处已改动，会返回冲突错误，需刷新后手动处理
- 文件名变更或发布状态切换前，会保留最近一次 `.bak` 备份

### 使用建议

- 先写标题、日期、slug、摘要，再写正文
- 点“保存”后再点“保存并预览”
- 新建页会强制以草稿创建；需要先完成创建，再在编辑页发布
- 如果要发布，直接点击“发布”即可，不需要先单独切换状态
- 如果收到冲突提示，先复制当前内容，再刷新页面处理

## 3. 手工草稿示例

```mdx
---
title: "我的第一篇文章"
date: "2026-03-23"
slug: "my-first-post"
excerpt: "一句话描述文章核心内容。"
tags: ["工程", "工具"]
published: false
---

开始写正文……

## 第一章

内容在这里。
```

> `published: false` = 草稿。草稿不会出现在 `/blog`，只能在作者侧看到。

## 4. 发布方式

### 在线工作台发布

- 打开 `/dashboard/write/[slug]`
- 编辑页右侧会显示当前发布状态
- 直接点击“发布”或“取消发布”按钮，系统会走同一保存链路提交
- 保存成功后，文章即可被 `/blog` 读取

### 手工发布

打开文件，将 `published: false` 改为 `published: true`，保存。

```bash
bash scripts/release-check.sh
sudo systemctl restart personal-blog
```

新文章约 10 秒内出现在 `/blog`。

## 5. Frontmatter 完整字段

```yaml
---
title: "文章标题"
date: "yyyy-MM-dd"
slug: "url-slug"
excerpt: "摘要"
tags: ["标签1", "标签2"]
published: false
cover: "/images/cover.jpg"
---
```

### 约束规则

| 规则 | 说明 |
|------|------|
| `slug` 与文件名中的 slug 必须一致 | 如 `2026-03-23_my-post.mdx` → `slug: my-post` |
| `date` 格式为 `yyyy-MM-dd` | 严格校验，写错会变成待修复或保存失败 |
| `published` 无默认值 | 缺失即为 invalid，必须显式声明 |
| `excerpt` 必填 | 用于列表页展示和 SEO 描述 |

---

## 6. 文章状态说明

| 状态 | 显示位置 | 条件 |
|------|---------|------|
| 已发布 | `/blog`、首页精选 | `published: true` + frontmatter 验证通过 |
| 草稿 | `/dashboard`、`/dashboard/write` | `published: false` + frontmatter 验证通过 |
| 待修复 | `/dashboard` | frontmatter 验证失败 |

---

## 7. Dashboard 内容工作台

地址：`/dashboard`（需登录授权）

功能：
- 查看所有文章及状态统计
- 进入在线写作工作台 V2
- 快速定位待修复文章
- 编辑草稿或已发布文章

Dashboard 通过 `getAllPostEntries()` 读取所有文件，而 `/blog` 只显示已发布内容。
