# 文章发布流程 — 从写文章到上线

> V1 版本，基于文件驱动的 MDX 工作流。
> 写作门槛：能写 Markdown，会用文本编辑器即可。

---

## 1. 三步上线

### Step 1：写草稿

在 `content/posts/` 下新建文件：

```
yyyy-MM-dd_slug.mdx
```

文件名格式：`日期_slug`，日期与 frontmatter 中的 `date` 必须一致。

**草稿文件示例：**

```mdx
---
title: "我的第一篇文章"
date: "2026-03-23"
slug: "my-first-post"
excerpt: "一句话描述文章核心内容。"
tags: ["工程", "工具"]
published: false
---

开始写正文……

## 第一章

内容在这里。
```

> `published: false` = 草稿。草稿不会出现在 `/blog`，只能在 `/dashboard` 看到。

### Step 2：本地预览

```bash
cd /home/cjl/projects/personal-blog
npm run dev
# 打开 http://localhost:3000/dashboard 预览
```

进入 `/dashboard`（需 GitHub 登录 + 白名单），查看文章状态：
- **草稿**（黄）：文章有效，但未发布
- **待修复**（红）：frontmatter 验证失败，按提示修改

### Step 3：发布

打开文件，将 `published: false` 改为 `published: true`，保存。

```bash
# 检查
bash scripts/release-check.sh

# 发布（重启服务）
sudo systemctl restart personal-blog
```

新文章约 10 秒内出现在 `/blog`。

## 2. Frontmatter 完整字段

```yaml
---
title: "文章标题"           # 必填，页面 <h1>
date: "yyyy-MM-dd"          # 必填，格式严格校验
slug: "url-slug"            # 必填，URL 路径，文件名中的 slug 必须与此一致
excerpt: "摘要"             # 必填，用于列表页和 SEO 描述
tags: ["标签1", "标签2"]    # 必填，可为空数组
published: false             # 必填，true = 发布，false = 草稿
cover: "/images/cover.jpg"  # 可选，文章封面图
---
```

### 约束规则

| 规则 | 说明 |
|------|------|
| `slug` 与文件名中的 slug 必须一致 | 如 `2026-03-23_my-post.mdx` → `slug: my-post` |
| `date` 格式为 `yyyy-MM-dd` | 严格校验，写错会变成"待修复" |
| `published` 无默认值 | 缺失即为 invalid，必须显式声明 |
| 所有文章必须有 `excerpt` | 用于列表页展示和 `<meta name="description">` |

## 3. 文章状态说明

| 状态 | 显示位置 | 条件 |
|------|---------|------|
| **已发布** | `/blog`、首页精选 | `published: true` + frontmatter 验证通过 |
| **草稿** | 仅 `/dashboard` | `published: false` + frontmatter 验证通过 |
| **待修复** | 仅 `/dashboard` | frontmatter 验证失败（含具体错误） |

---

## 4. Dashboard 内容工作台

地址：`/dashboard`（需登录授权）

功能：
- 查看所有文章及状态统计
- 快速定位需要修复的文章
- 点击"查看 →"预览已发布的文章

Dashboard 通过 `getAllPostEntries()` 读取所有文件（含草稿和无效文件），而 `/blog` 列表页只显示已发布的文章。

## 5. 快速参考

```bash
# 新建草稿
touch content/posts/$(date +%Y-%m-%d)_my-slug.mdx

# 验证 frontmatter
npm run lint

# 本地预览
npm run dev

# 发布前检查
bash scripts/release-check.sh

# 重启服务
sudo systemctl restart personal-blog
```

### 文件命名

```
yyyy-MM-dd_slug.mdx
  └─ date 必须与 frontmatter.date 一致
  └─ slug 必须与 frontmatter.slug 一致
```
