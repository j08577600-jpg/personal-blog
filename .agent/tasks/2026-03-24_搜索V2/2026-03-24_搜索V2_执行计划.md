## 目标

本功能完成后，用户在 `/search` 搜索时可以通过 `?q=` 分享和恢复搜索状态，刷新不丢失输入与结果；搜索结果的数据源只包含结构有效的公开内容；正式搜索范围从 `posts` 扩展到 `projects`，并为 `notes / reading-list` 留出清晰、低风险的后续接入路径。

## 决策日志

- Decision: V2 正式索引范围先做 `posts + projects`
  原因：两类内容展示结构最接近，用户价值最高，实施风险最低；`notes + reading-list` 先做映射预案更稳
  日期/作者：2026-03-24 / planner-codex

- Decision: 搜索状态以 URL 为唯一外部状态源，不引入 localStorage
  原因：可刷新、可分享、可前进后退，且不引入多标签页状态漂移
  日期/作者：2026-03-24 / planner-codex

- Decision: 中文搜索 V2 只做 L1 级轻量优化
  原因：当前主要痛点不是分词能力，而是状态恢复和索引范围；避免过度设计
  日期/作者：2026-03-24 / planner-codex

- Decision: 构建期索引脚本切换为 `tsx scripts/build-search-index.ts`。
  原因：需要直接复用 TypeScript 的内容读取与搜索映射逻辑，避免在 Node ESM 下重复维护一份 JS 版索引口径。
  日期/作者：2026-03-24 / builder-codex

- Decision: 搜索 query 的历史链路改为显式使用浏览器 `history.pushState`，不再依赖 App Router 对 query-only 导航的 history 行为。
  原因：P4 回修实测发现 `router.push` 在 `/search?q=a -> /search?q=b` 场景下未形成可回退历史栈，无法满足“后退/前进恢复”验收项；改为显式写 history 后，链路可稳定恢复。
  日期/作者：2026-03-24 / builder-codex

## 实施步骤

### Step 1: 修正索引数据源口径
- 文件：`scripts/build-search-index.mjs`、`src/lib/posts.ts`（如需）、`src/lib/projects.ts`（如需）
- 目标：确保搜索索引只来自“公开且结构有效”的内容，不再绕过现有内容校验
- 修改要点：
  - 不再仅凭 `published === true` 就进入索引
  - `posts` 必须复用现有有效内容读取口径，排除 invalid post
  - `projects` 接入时也遵循“仅公开内容”原则
- 预期结果：无效文章不再出现在 `public/search-index.json`

### Step 2: 升级统一搜索索引模型并纳入 projects
- 文件：`scripts/build-search-index.mjs`、`src/lib/search.ts`（可新增）、`src/components/search-box.tsx`
- 目标：把索引条目升级为统一结构，并正式支持 `posts + projects`
- 修改要点：
  - 为每个条目补齐 `type / href / section / title / excerpt / tags / date`
  - 搜索结果渲染不再硬编码 `/blog/${slug}`，改为使用 `href`
  - 结果列表展示中增加类型信息，避免文章/项目混淆
- 预期结果：`public/search-index.json` 为统一结构数组；页面能渲染 posts 与 projects 混合结果

### Step 3: 实现 URL 查询参数持久化与恢复
- 文件：`src/app/search/page.tsx`、`src/components/search-box.tsx`
- 目标：支持 `/search?q=`，刷新不丢失状态
- 修改要点：
  - 首屏读取 URL query 作为初始搜索词
  - 输入变化 debounce 后同步更新 URL
  - 为空时移除 `q` 参数
  - URL 变化时组件状态可回填，兼容前进/后退
- 预期结果：输入、URL、结果三者保持一致

### Step 4: 完成中文搜索 L1 轻量优化
- 文件：`src/components/search-box.tsx`、`scripts/build-search-index.mjs` 或 `src/lib/search.ts`
- 目标：在不引入重型依赖的前提下提升中文短词体验
- 修改要点：
  - 对 query 与索引文本做统一轻量规范化
  - 为 `title / tags / excerpt` 设置更合理权重
  - 复核 Fuse 阈值，避免过宽匹配
- 预期结果：中文短 query 的相关性较 V1 更稳，不出现明显无关命中增多

  - 构建成功
