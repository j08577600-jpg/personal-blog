---

## 1. 目标

建立以 `content/posts/*.mdx` 为单一内容源的文件驱动博客系统，包括：
1. MDX 文件读取与渲染管道
2. 博客列表页（支持按 `published` 过滤）
3. 文章详情页（读取 slug 并渲染 MDX 内容）
4. 与现有 `PostCard`、`Post` 类型兼容

---

## 2. 背景

博客项目（Next.js 16 + Tailwind + NextAuth）已完成基础架子，但内容层仍使用 `src/lib/posts.ts` 内的硬编码 mock 数据，与规范中定义的 V1 原则（"内容源只认 `content/posts/*.mdx`"）不符。

现有代码已预留 `PostCard` 组件和列表/详情页骨架，只需替换数据层。

## 4. 技术方案

### 4.1 依赖选型

| 包 | 版本 | 用途 |
|----|------|------|
| `gray-matter` | ^4 | 解析 MDX frontmatter |
| `next-mdx-remote` | ^5 | RSC 友好 MDX 渲染（App Router） |
| `reading-time` | ^1 | 计算阅读时长 |

> 不选用 `@next/mdx`：它要求 `next.config.ts` 中配置并受 Next.js 编译管道约束，对文件系统动态读取不友好。`next-mdx-remote` 支持 Server Component 中手动传入 MDX 源码，灵活性更高。

### 4.2 文件名规范（继承规范）

```
content/posts/yyyy-MM-dd_slug.mdx
```

slug 部分使用连字符（kebab-case），如 `2026-03-23_my-first-post.mdx`。

### 4.3 Frontmatter Schema

```yaml
---
title: 文章标题
date: yyyy-MM-dd
slug: url-slug
excerpt: 文章摘要
tags: [标签1, 标签2]
published: true        # true = 发布，false = 草稿（列表页不展示）
cover: /images/xxx.jpg  # 可选，暂无渲染逻辑
---
```

> 与 `docs/技术框架与代码规范.md` 保持一致。

### 4.4 `Post` 类型变更

```ts
// 之前
content: string[]  // 段落数组（mock 数据结构）

// 之后
content: string    // 原始 MDX 字符串
```

影响范围：
- `PostCard`：只用 frontmatter 字段，**不受影响**
- `src/app/blog/[slug]/page.tsx`：改用 `MDXRemote` 渲染，**需调整**
- `src/app/page.tsx`：只用 frontmatter 字段，**不受影响**

### 4.5 `src/lib/posts.ts` 改造

```
getPosts()     → 扫描 content/posts/*.mdx，过滤 published:true，按 date 倒序
getPostBySlug(slug) → 读取对应 .mdx 文件，返回 Post | null
```

- 使用 Node.js `fs`（`readdir`, `readFile`）同步读取
- 路径基于 `process.cwd()` + `content/posts/`
- `readingTime` 在 `getPosts` / `getPostBySlug` 内由 `reading-time` 包计算
- 错误处理：文件不存在或 frontmatter 缺失时返回 `null`

### 4.6 文章详情页渲染

`src/app/blog/[slug]/page.tsx` 改为：

```tsx
import { MDXRemote } from "next-mdx-remote/rsc";
import { getPostBySlug } from "@/lib/posts";

// generateStaticParams、getPostBySlug 调用不变
// 渲染：
<MDXRemote source={post.content} />
```

> `MDXRemote` 是 React Server Component，App Router 原生支持，无需 `"use client"`。

## 6. 风险与取舍

### 6.1 风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| MDX 内容含未声明的 import/require | 低 | 中 | V1 不支持 MDX 内嵌 import，说明文档中注明 |
| `process.cwd()` 在构建时与运行时路径不一致 | 低 | 高 | 统一使用 `process.cwd()`，文档注明构建/运行要求 |
| `content/posts/` 目录不存在时构建报错 | 低 | 中 | `getPosts()` 内部做 try/catch，空目录时返回 `[]` |
| `reading-time` 对中文分词不准 | 低 | 低 | 可后续替换为 `reading-time` + 自定义汉字分词 |

### 6.2 取舍

| 决策 | 理由 |
|------|------|
| 继续用 `gray-matter` 而非手写 frontmatter 解析 | 稳定、广泛使用，减少边界 case 处理 |
| `content: string[]` → `content: string` 破坏类型 | 这是 V1 mock 数据遗留问题，修正是正确的 |
| 不引入数据库层 | 严格遵循 V1 规范 |
| 不引入 MDX 自定义组件 | V1 保持简单，够用即可 |
