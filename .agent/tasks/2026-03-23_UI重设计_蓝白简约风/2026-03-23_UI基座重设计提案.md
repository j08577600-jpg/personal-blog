## 一、现状分析

### 当前设计系统
| 维度 | 当前值 |
|------|--------|
| 主背景（亮色） | `#f7f7f5`（米灰） |
| 主背景（暗色） | `#090909`（近黑） |
| 主文字（亮） | `#111111` |
| 主文字（暗） | `#f3f3f3` |
| 强调色 | 纯黑/纯白（opacity 驱动） |
| 蓝色 | **无** |
| Container 宽度 | `max-w-5xl`（列表页）/ `max-w-3xl`（文章页） |
| 圆角 | `rounded-2xl`（卡片）/ `rounded-full`（按钮/标签） |
| 边框 | `border-black/5` ~ `border-black/12` |
| 字体 | Geist Sans + Geist Mono |
| Header | 白色半透明 + backdrop-blur |
| Footer | 极简文字排版，无装饰 |
| 暗色适配 | 纯色反转，无蓝色过渡 |

### 核心问题
1. **无视觉锚点**：全站无品牌色，蓝色系为零，视觉上偏"极简灰色调"而非"现代蓝白"
2. **字号层级偏小**：lead text 14px / body 14px，与 5xl 标题搭配略显拥挤
3. **暗色体验不够完整**：dark mode 仅做颜色反转，蓝白系统需同时考虑暗色变体
4. **组件一致性**：PostCard、按钮、标签的间距/圆角/hierarchy 未形成系统
5. **无页面节奏感**：首页 sections 间距不统一，部分区域视觉密度过高

## 二、目标设计系统

### 2.1 颜色体系（蓝白色调）

| Token | 色值 | 用途 |
|-------|------|------|
| `--brand-50` | `#EFF6FF` | hover 背景、选中态 |
| `--brand-100` | `#DBEAFE` | tag 背景、badge |
| `--brand-500` | `#3B82F6` | 主强调（链接 hover、主 CTA） |
| `--brand-600` | `#2563EB` | 主 CTA 背景色 |
| `--brand-700` | `#1D4ED8` | 主 CTA hover |
| `--brand-900` | `#1E3A5F` | 深色文字配白背景（极少见） |
| `--background` | `#FFFFFF` | 主背景（由米灰改为纯白） |
| `--surface` | `#F8FAFC` | 卡片/侧边栏背景（ slate-50） |

### 2.2 间距体系
沿用 Tailwind 默认间距（1rem=16px），在组件层面统一规则：
- **Section 间距**：`py-20`（首页主区域）/ `py-16`（次要区域）
- **Card 内边距**：`p-6`（小卡）/ `p-8`（大卡/侧边栏）
- **Grid gap**：`gap-5`（列表）/ `gap-10`（首页双栏）
- **元素间距**：遵循 4px 基准（gap-1=4px），避免奇数间距
- **容器水平内边距**：`px-4 sm:px-6`（移动端至桌面端统一）

### 2.3 容器宽度
| 页面类型 | 宽度 | 备注 |
|----------|------|------|
| 首页 hero | `max-w-4xl` | 居左，留白感更强 |
| 博客列表/标签 | `max-w-4xl` | 比当前 max-w-5xl 稍窄，阅读更集中 |
| 文章页 | `max-w-2xl` | 从 max-w-3xl 收窄，提升阅读舒适度 |
| 仪表盘 | `max-w-3xl` | 居中，功能区不宜过宽 |

### 2.4 页面节奏
- **首页 Hero**：上下留白 `py-24`，标题与正文间距增大至 `mt-8`
- **精选文章区**：与 Hero 间距 `mt-24`，增加视觉呼吸感
- **博客列表**：文章间距 `gap-6`（从 gap-5 略微拉开）
- **文章页正文**：内容区上下 `py-16`，back-link 改为顶部左侧固定

### 2.5 Header 规范
```
高度：py-4（56px 总高）
背景：bg-white/90 backdrop-blur（亮）/ bg-slate-950/90 backdrop-blur（暗）
Logo：品牌蓝 + 品牌名，font-weight: 600，size: text-sm
导航项：text-sm text-slate-500 → hover: text-slate-900（亮）/ text-slate-100（暗）
分隔线：border-b border-slate-200（亮）/ border-slate-800（暗）
CTA 按钮：GitHub 登录 → 蓝色填充按钮，风格统一
```

### 2.6 Footer 规范
```
背景：border-t border-slate-200（亮）/ border-slate-800（暗）
Logo 行：左侧 Logo + 右侧社交链接（GitHub / RSS）
链接行：博客 / 关于 / 标签（text-sm text-slate-500）
版权行：text-xs text-slate-400
布局：sm:flex-row，支持移动端堆叠
```

### 2.7 PostCard 组件
```
容器：rounded-2xl border border-slate-200 bg-white（亮）
     dark: border-slate-800 bg-slate-900（暗）
     transition: hover:shadow-md hover:border-slate-300（亮）/ hover:border-slate-700（暗）
内边距：p-6
元数据行：text-xs text-slate-400 uppercase tracking-wide（替换 text-sm）
标题：text-xl font-semibold text-slate-900（亮）/ text-slate-100（暗）
摘要：text-sm text-slate-600（亮）/ text-slate-400（暗）leading-7
标签：rounded-full bg-slate-100 text-slate-600（亮）
     dark: bg-slate-800 text-slate-300（暗）
     hover: bg-blue-100 text-blue-600（亮）→ 可选蓝色 tag 交互
```

### 2.8 按钮系统
| 类型 | 亮色 | 暗色 |
|------|------|------|
| Primary CTA | `bg-blue-600 text-white hover:bg-blue-700 rounded-full px-5 py-3 text-sm font-medium` | `bg-blue-500 hover:bg-blue-400 text-white` |
| Secondary | `border border-slate-300 text-slate-700 hover:border-slate-400 hover:bg-slate-50 rounded-full` | `border border-slate-700 text-slate-200 hover:bg-slate-800` |
| Ghost | `text-slate-600 hover:text-slate-900 hover:bg-slate-100 rounded-lg px-3 py-2` | `text-slate-400 hover:text-slate-100 hover:bg-slate-800` |

### 2.9 Tag/标签系统
```
默认态：rounded-full px-2.5 py-1 text-xs
        bg-slate-100 text-slate-600（亮）
        bg-slate-800 text-slate-300（暗）
        border border-transparent
Hover 态：bg-blue-100 text-blue-700（亮）
         bg-blue-900/30 text-blue-300（暗）
         border border-blue-200（亮）/ border-blue-800（暗）
```

### 2.10 文章正文渲染
- 标题层级：`h2 text-2xl font-semibold text-slate-900 mt-12 mb-4` / `h3 text-xl`
- 正文：`text-base leading-7 text-slate-700`（行高从 6 增到 7）
- 引用块：`border-l-4 border-blue-400 bg-blue-50 pl-4 py-1 rounded-r`
- 代码块：`bg-slate-900 text-slate-100 rounded-xl p-4 overflow-x-auto`
- 链接：`text-blue-600 underline underline-offset-2 hover:text-blue-700`

## 四、风险与约束

### 高风险
1. **暗色模式一致性**：blue 色系在暗色下的对比度需 WCAG AA 级以上，blue-500/slate-950 组合对比度约 5.3:1（OK），但 blue-400 用于文字时需谨慎
2. **现有文章内容**：CSS 变量改动会影响所有 MDX 渲染层（article-renderer），需确保 prose 颜色覆盖正确

### 中风险
3. **GitHub 登录按钮**：当前 HeaderAuthControls 用 border-button 风格，统一为蓝色 Primary CTA 后需确认视觉层级
4. **首页 Hero 宽度**：`max-w-4xl` 从 `max-w-3xl` 收窄的方案需验证内容截断情况
5. **部署预览**：改动范围大，建议分批上线而非一次性替换

### 低风险
6. **向后兼容**：CSS 变量系统替换不影响 JS/TS 逻辑
7. **SEO 影响**：颜色变化不影响 metadata，但 Hero 标题字号变化可能轻微影响 CLS

## 六、给 Planner 的建议

### 是否拆 P1/P2
建议 **合并为单个 P2**，原因是：
- 改动集中于 CSS 变量 + Tailwind class 替换
- 无跨技术栈依赖（不涉及 auth/MDX/DB）
- 可分 Phase 执行，每 Phase 独立可验收

### 验收标准建议
1. Lighthouse Accessibility ≥ 90
2. 所有页面 light/dark mode 双色截图通过审查
3. `npm run build` 通过，无 TS 报错
4. PostCard、Button、Tag 在三个核心页面表现一致
