# UI 重设计 · 蓝白简约风 · 方案设计

> 日期：`2026-03-23`
> 任务：UI 重设计 · 蓝白简约风
> 负责人：planner-minimax
> 状态：**P1 · 方案设计完成** → 等待 P2 执行计划

---

## 目标

以「现代简约·蓝白·内容优先」为设计语言，对全站 UI 进行系统性重设计。统一视觉锚点、修复暗色模式、提升阅读体验，同时保持现有信息架构和功能逻辑不变。

---

## 背景

**现状痛点：**
1. **无视觉锚点** — 全站无品牌色，蓝色为零，视觉偏「极简灰色调」而非「现代蓝白」
2. **暗色体验残缺** — 仅做颜色反转，蓝白色系未覆盖暗色变体
3. **组件一致性差** — PostCard、按钮、标签的间距/圆角/层级未形成系统
4. **页面节奏感弱** — Hero 与内容区间距不统一，视觉密度忽高忽低
5. **正文排版粗放** — 字号层级偏小，行高偏紧，阅读舒适度不足

**时机：** 现有功能稳定，内容结构不变，是重塑视觉系统的最佳窗口。

## 技术选择

### 设计系统方案

采用 **CSS Custom Properties + Tailwind v4 `@theme`** 方案：

```css
/* globals.css */
@theme {
  --color-bg-primary: #f8fafc;
  --color-bg-surface: #ffffff;
  --color-bg-subtle: #f1f5f9;
  --color-text-primary: #0f172a;
  --color-text-secondary: #475569;
  --color-text-muted: #94a3b8;
  --color-accent: #2563eb;
  --color-accent-hover: #1d4ed8;
  --color-accent-subtle: #eff6ff;
  --color-border: #e2e8f0;
}
```

**优势：**
- 语义化命名（--accent 而非 --blue-600），组件层易读
- Tailwind v4 `@theme` 直接映射为 utility class，无需 JS config
- dark mode 用 `[data-theme="dark"]` 或 `dark:` class 覆盖，无需 extra JavaScript
- 颜色体系覆盖亮/暗双态，无遗漏

**暗色变体（dark class）：**

| Token | 亮色 | 暗色 |
|-------|------|------|
| `--color-bg-primary` | `#f8fafc` | `#0c1222` |
| `--color-bg-surface` | `#ffffff` | `#111827` |
| `--color-bg-subtle` | `#f1f5f9` | `#1e293b` |
| `--color-text-primary` | `#0f172a` | `#f1f5f9` |
| `--color-text-secondary` | `#475569` | `#94a3b8` |
| `--color-text-muted` | `#94a3b8` | `#64748b` |
| `--color-accent` | `#2563eb` | `#60a5fa` |
| `--color-accent-hover` | `#1d4ed8` | `#93c5fd` |
| `--color-accent-subtle` | `#eff6ff` | `#1e3a5f` |
| `--color-border` | `#e2e8f0` | `#1e293b` |

**替代方案 A：纯 Tailwind config**
- 劣势：v4 鼓励 `@theme`，config 路径在 v4 中已不推荐
- 结论：不选

**替代方案 B：Radix UI / shadcn/ui 组件库**
- 劣势：引入 50+ 依赖，与现有简洁结构冲突，改造量巨大
- 结论：不选

### 必须安装的依赖

```bash
npm install @tailwindcss/typography
```

提供 `prose` / `prose-lg` 类，规范化 MDX 正文排版（行高 1.8、段落间距 1.5em）。

### 可选图标库

```bash
npm install lucide-react
```

用于标签页、日期行等处的 SVG 图标（如 Calendar、Tag 图标）。**P1 阶段可选，P2 实施时按需引入**。

### 组件改动策略

- 所有改动仅涉及 Tailwind class 替换 + CSS 变量引用
- 不改变 JSX 结构（除 Hero 竖条需额外 `<span>` 包裹）
- 不改变 props 接口
- 可中断、可回滚（git 分支隔离）

## 风险与取舍

### 高风险

| 风险 | 缓解策略 |
|------|---------|
| 暗色模式覆盖不完整，页面出现「白斑」 | Phase 0 必须完成 globals.css 双态验证；每 Phase 提交前截图审查暗色模式 |
| Tailwind v4 `@theme` 与现有构建系统不兼容 | 先在 dev 环境验证，确认后合并；保留 revert 分支 |
| `@tailwindcss/typography` prose 类覆盖现有 MDX 样式，产生冲突 | article-renderer.tsx 中使用 `prose prose-lg dark:prose-invert`，同时用 Tailwind 变量覆盖冲突色值 |

### 中风险

| 风险 | 缓解策略 |
|------|---------|
| 蓝色饱和度过高刺眼 | 固定使用 `#2563eb`（适中蓝），不用亮蓝 `#3b82f6` 作为主文字色 |
| 暗色深蓝黑（`#0c1222`）与博客气质不搭 | 备选 `#0f172a`（slate-950）更稳重；可在 Phase 0 测试后拍板 |
| 改动文件多（12 个），跨度大，容易引入回归 | 每 Phase 独立可验收；每 Phase 提交前必须 `npm run lint && npm run build` |
| 首页 Hero `max-w-4xl` 从 `max-w-3xl` 扩宽，内容截断 | Phase 2 上线前本地验证，未通过不合并 |

### 低风险

| 风险 | 缓解策略 |
|------|---------|
| CSS 变量覆盖影响 MDX prose 渲染 | 使用 `prose-headings:text-[--color-text-primary]` 等 Tailwind prose 插件选项显式覆盖 |
| GitHub 登录按钮风格与新按钮系统不一致 | Phase 1 包含 header-auth-controls.tsx 和 github-signin-button.tsx 的样式对齐 |
| 性能/CLS 影响 | Hero 标题字号变化轻微，建议保持现有字体大小，验收时跑 Lighthouse |

## 品牌语调与文案规范（Copy as First-Class）

> 设计不仅是颜色和布局，文案语气和标签措辞同样影响用户体验。本节作为设计系统的软性维度，与配色同等重要。

### 文案基调

- **整体语气：** 克制、专业、有温度。不夸张、不讨好、不冷冰冰。
- **行动号召：** 用动词开头，简洁有力（如「查看文章」而非「欢迎查看我的所有文章」）。
- **标签措辞：** 使用中文名词（技术栈名直接英文，如 TypeScript、Next.js）。

### 微文案规范

| 场景 | 当前文案 | 目标文案 | 说明 |
|------|---------|---------|------|
| 首页 Hero 副标题 | （需确认） | 「记录技术思考，分享前端与全栈实践」 | 明确价值主张，交代内容方向 |
| 博客列表页标题 | Blog | 博客 | 更简洁 |
| 博客列表页计数 | （无） | 「共 N 篇文章」 | 标签页已有，博客列表补充 |
| 文章详情页 Back link | ← Back | ← 返回博客 | 语义更清晰 |
| 标签胶囊 hover | 无变化 | 蓝底白字 | 与视觉系统同步 |

| 空状态 | （需确认） | 「暂无文章，敬请期待」 | 符合克制语气 |

### 文案验收维度

1. **一致性** — 全站标签语气统一（技术词大小写、标点使用）
2. **清晰性** — 无歧义，无语法错误，无过度口语化
3. **克制性** — 不过度营销，不过度解释
4. **SEO 友好** — title / meta description / heading 层级语义正确

## 结论

本方案整合 Task A（UI基座重设计）和 Task B（蓝白简约风方案设计），作为单一功能推进。

**核心设计决策：**
1. 采用语义化 CSS 变量 + Tailwind v4 `@theme` 构建蓝白配色系统，亮/暗双态完整覆盖
2. PostCard 采用阴影 hover 体系，减少边框视觉噪音
3. Hero / 标题区统一左侧 4px 蓝色竖条锚点，建立全站视觉识别
4. 标签从边框小胶囊升级为 accent-subtle 背景胶囊
5. 文章详情页收窄至 max-w-2xl，引入 typography prose 提升阅读体验
6. 分 4 Phase 渐进上线，每 Phase 独立可验收，最大限度控制风险

**是否进入下一阶段：** ✅ P1 完成，可进入 P2 执行计划阶段。
