## 非阻塞问题（Non-blocking Improvements）

### 1. `generateMetadata` 中冗余调用 `getPosts()`（低优先级）

**位置：** `src/app/blog/tags/[tag]/page.tsx` — `generateMetadata` 函数

**问题：** `generateMetadata` 中调用了 `getPosts().filter(...)` 来计算 `posts.length`，这与页面组件中重复调用了 `getPosts()`（两次：一次在 `allTags` 计算，一次在 `posts` 过滤）。文件 I/O 被调用了 3 次。

**改进建议：** `posts.length` 可从 `getAllTags().find()` 返回的 `count` 字段获取：

```typescript
// generateMetadata 中：
const tagEntry = allTags.find((t) => t.tag === decodedTag);
if (!tagEntry) return { title: "标签未找到" };
const url = `${siteConfig.siteUrl}/blog/tags/${encodeURIComponent(tag)}`;
return {
  title: `标签：${decodedTag}`,
  description: `浏览所有带有「${decodedTag}」标签的文章，共 ${tagEntry.count} 篇。`,
  alternates: { canonical: url },
};
```

**优先级：** 低 — V1 体量下文件 I/O 可忽略；后续加 LRU 缓存后问题自然消失。

---

### 2. `posts.ts` 中的遗留 PostCard 定义（不影响）

**说明：** 执行计划"意外与发现"节已记录此事：`src/lib/posts.ts` 末尾（`getAllTags()` 之后）原有一个冗余的 PostCard 定义。Builder 未动此遗留代码，保留 `src/components/post-card.tsx` 为唯一真实来源。行为正确，但 `posts.ts` 中有 ~25 行无用代码。

**改进建议：** 后续清理时删除 `posts.ts` 中的 PostCard 副本。

**优先级：** 低 — 不影响功能，不在本次 V1 范围内。

---

### 3. `params` 解构后缺少 Promise 展开警告（TS strict）

**位置：** `src/app/blog/tags/[tag]/page.tsx` — `Props` 接口和 `generateMetadata`/`TagPage` 函数

**现状：** `params` 声明为 `Promise<{ tag: string }>`，但使用时直接 `await params` 再解构。Next.js 16 App Router 文档中，params 在某些版本下已可直接解构（不再需要 Promise 包装）。

**说明：** 当前代码行为正确（build 通过），但若 Next.js 后续版本移除 params Promise 包装，需同步更新。

**改进建议：** 保持观察，后续 Next.js 版本更新时同步调整。

**优先级：** 低 — Next.js 版本锁定，需同步跟进。

## 二审详细说明

### 1. 导航入口（已修复 ✅）

**问题：** `src/lib/site.ts` 的 `navigation` 数组中缺少 `/blog/tags` 入口。作为博客的核心发现功能，标签页应该在全局导航中可被直接访问，否则只能通过文章卡片中的标签跳转，路径过深。

**修复：** 在 `navigation` 中添加 `{ href: "/blog/tags", label: "标签" }`。

**验证：** `npm run build` 通过，全局导航组件 `SiteHeader` 会自动渲染新入口。

---

### 2. 标签索引页底部回流（已修复 ✅）

**问题：** `src/app/blog/tags/page.tsx` 底部缺少回流链接。用户浏览完标签列表后，只能依赖浏览器返回或手动跳转，无法直接"回到博客"。

**修复：** 在标签列表下方添加"← 返回博客"链接，风格与 `/about` 页面的 CTA 保持一致（`text-black/55 hover:text-black`）。

**文案选择：** 使用"返回博客"而非"返回首页"，因为用户在标签页的上下文中，下一步更可能是继续浏览博客文章而非跳转首页。

---

### 3. 文案与风格审查

**检查项：**
- 标题 "按标签浏览"：简洁、清晰，与博客整体克制风格一致。
- 空状态 "暂无标签" / "该标签下暂无已发布文章"：语气平和、友好，与博客其他页面的空状态风格统一。
- 标签链接中的 `(count)` 数量标注：视觉权重适中（`text-xs text-black/35`），不抢夺标签名的主体地位。
- 标签详情页的 "标签：{tag}" 标题：使用中文冒号，与博客的 metadata 模板 `%s · {siteName}` 配合时格式清晰。

**结论：** 文案质量高，无需修改。

---

### 4. 样式一致性审查

**检查项：**
- `max-w-5xl` + `px-6` + `py-20`：与 `/blog` 和 `/about` 页面完全一致。
- `text-4xl font-semibold tracking-tight sm:text-5xl`：与 `/blog` 和 `/about` 的 H1 样式一致。
- 标签 pill 样式（`rounded-full border border-black/8 px-4 py-2`）：与 PostCard 中的标签样式逻辑一致。
- Dark mode：使用 `dark:` variant 正确，颜色值（如 `border-white/10`）与博客其他元素（如 PostCard）保持一致。

**结论：** 样式一致性良好。
