### 核心变更

| `src/app/blog/tags/[tag]/page.tsx` | 新增 | +64 行：标签详情页（Server Component） |
| `src/components/post-card.tsx` | 修改 | `span` → `Link`，hover 样式 |
| `src/lib/site.ts` | 修改 | navigation 数组新增 `/blog/tags` 入口（二审修复） |

### 关键设计决策

1. **方案 A（推荐）：** 在 `posts.ts` 中新增 `getAllTags()`，两个页面均为 Server Component
2. **URL 策略：** 使用原始 tag 字符串（`encodeURIComponent` 编码），不做 slugify，避免大小写/空格歧义
3. **PostCard：** 标签 `<span>` → `<Link>` 内联，无需 `"use client"`，样式复用 + hover 效果
4. **generateMetadata 优化（post-P5）：** `generateMetadata` 中使用 `tagEntry.count` 替代冗余 `getPosts()` 调用
