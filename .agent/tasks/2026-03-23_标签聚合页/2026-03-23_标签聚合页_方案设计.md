# 标签聚合页 方案设计

> 日期：2026-03-23
> 任务：标签聚合页
> 负责人：planner-codex

## 目标

为博客添加标签索引页和标签详情页，让读者按主题浏览文章。

## 背景

博客已有 tags 字段，但无法按标签筛选文章。用户需要一个入口页面列出所有标签，以及每个标签对应的文章列表。

## 范围

### 做
- `getAllTags()` 和 `getPostsByTag()` 工具函数（posts.ts）
- `/tags` 页面：所有标签列表（带文章数）
- `/tags/[tag]` 页面：该标签下的文章列表
- 导航栏添加「标签」入口

### 不做
- 标签云/动态标签样式（留待后续）
- 标签管理后台（内容工作台范围外）
- 多标签筛选

## 技术选择

基于现有 posts.ts 的 `getPosts()`，在 lib 层添加两个纯函数：
- `getAllTags()` — 汇总所有已发布文章的标签，返回 `{ tag, count }[]`
- `getPostsByTag(tag: string)` — 返回指定标签的已发布文章

页面直接复用 `PostCard` 组件，保持视觉一致性。

## 风险与取舍

- 风险：标签数量多时列表过长 → V1 先做平铺，后续按数量排序
- 取舍：不单独做标签统计缓存，实时从文件系统读取（V1 规模足够）

## 结论

在 `src/app/tags/` 下添加两个页面，复用现有 posts.ts 工具函数和 PostCard 组件，快速实现标签聚合功能。
