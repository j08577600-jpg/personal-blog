## 一、设计理念

在现有克制、简洁的框架上，引入蓝白色调的系统性设计语言。

**核心原则：**
- 蓝 = 信任感、专业感、安静感；不抢内容视线
- 白/浅灰 = 干净的背景，留白的呼吸感
- 蓝白之间加适度灰度作为过渡，避免高饱和度刺眼
- 暗色模式：在深色底上保留蓝白对比（蓝作为强调色，黑白灰承载结构）

**不改变的内容：**
- 站点架构、路由结构
- 功能逻辑、交互流程
- 内容模型（MDX frontmatter 不变）
- 整体克制、不追求装饰感的设计哲学

## 三、通用设计语言

### 3.1 圆角系统

| 元素 | 当前 | 提案 |
|------|------|------|
| 页面容器 | `max-w-5xl px-6` | `max-w-4xl px-8`（内容更聚中，留白更大） |
| 卡片（PostCard） | `rounded-2xl` | `rounded-xl`（更利落，现代感） |
| 按钮 | `rounded-full` | 保留 `rounded-full`（胶囊形 CTA 与新风格不冲突） |
| 标签胶囊 | `rounded-full` | 保留，但背景改为 `--accent-subtle` + `--accent` 文字色 |
| 大标题面板 | 无特殊处理 | 左侧加 4px `--accent` 色竖条（视觉锚点） |

### 3.2 字体层级（调整间距和字重）

| 场景 | 提案调整 |
|------|---------|
| 页面标题（H1） | `text-4xl → text-5xl`，字重 700，增加 tracking-tight |
| Hero 段落 | `text-lg` → `text-xl`，增加行高到 1.75 |
| 正文 | 当前无明确 prose 设置，文章详情页应引入 `@tailwindcss/typography` 的 `prose` 类，确保行高 1.8、段落间距 1.5em |
| 辅助文字 | 当前用 `text-black/38` 等 opacity trick，改为语义化：`text-[--text-muted]` |

### 3.3 阴影（取代边框）

- 卡片 hover：去掉 border 变化，改为 `shadow-sm → shadow-md` + 轻微上移 `translateY(-2px)`
- 侧边栏（首页 aside）：用 `shadow-md` 替代厚边框感
- 保持：`border border-[--border]` 基础边框（1px），hover 时叠加阴影

### 3.4 动效

- 页面整体：transition 统一用 `duration-200 ease-out`
- 卡片 hover：`transition-all duration-200 hover:-translate-y-0.5 hover:shadow-md`
- 按钮 hover：`transition-colors duration-150`
- 禁止：过度动画、弹性 bounce、flash 效果

## 四、逐页设计提案

### 4.1 首页（`/`）

**当前状态：**
- Hero section：标题 + 描述 + 3 个 CTA 按钮
- 精选文章区：左侧文章列表，右侧 aside（站点定位）
- 布局：`lg:grid-cols-[1.2fr_0.8fr]`

**提案：**

| 区域 | 改动 |
|------|------|
| Hero 标题 | 左侧加 4px 蓝色竖条（H1 前加 `<span class="absolute left-0 w-1 h-full bg-[--accent] rounded-full"></span>`，配合 `relative pl-5`）|
| Hero 描述 | 字体加大至 `text-xl`，颜色改为 `--text-secondary` |
| 三个 CTA | 「查看文章」→ 蓝色填充按钮；其余两个 → 蓝色描边按钮（蓝白体系下更统一）|
| 精选文章标题 | 左侧加小竖条或 `.` 装饰，`<accent>` 色 |
| 右侧 aside | 背景改为 `--bg-subtle`，加 `shadow-sm`，不再是 border 感 |
| 整体间距 | 上 padding `py-16` → `py-24`，视觉节奏更舒展 |

**文件改动：** `src/app/page.tsx`

---

### 4.2 博客列表页（`/blog`）

**当前状态：**
- 页面标题 + 文章列表（纯 PostCard 堆叠）
- 无过滤、无排序

**提案：**

| 区域 | 改动 |
|------|------|
| 页面标题 | 同首页 Hero 风格，左侧加蓝色竖条 |
| 文章数量 | 标题下方加 `<p class="text-[--text-muted]">共 N 篇文章</p>` |
| PostCard | hover 时阴影效果取代纯 border 变化 |
| 标签胶囊 | 背景 `--accent-subtle`，文字 `--accent` |
| 文章间分隔 | 去掉 `<hr>` 分隔线，用卡片间距（gap-5）和不同 hover 阴影状态区分 |
| 底部 | 加载更多或分页（P1 阶段保持无限滚动或"查看更多"按钮，非分页器）|

**文件改动：** `src/app/blog/page.tsx`、`src/components/post-card.tsx`

---

### 4.3 文章详情页（`/blog/[slug]`）

**当前状态：**
- 简单 back link + 标题 + 日期 + 标签 + MDX 内容
- max-w-3xl，内容较宽

**提案：**

| 区域 | 改动 |
|------|------|
| 内容宽度 | `max-w-3xl` → `max-w-2xl`（阅读线长更舒适，约 65 字符）|
| back link | 改为蓝色文字 `text-[--accent]`，hover 下划线 |
| 标题下方 | 加 `<hr class="border-[--border] mb-8">` 分隔线 |
| 日期/阅读时间 | 文字改为 `--text-muted`，加阅读时间图标（📖 或 SVG）|
| 标签区 | 移到标题上方（先看到主题再决定是否阅读），背景 `--accent-subtle` 胶囊 |
| MDX 正文 | 应用 `@tailwindcss/typography` prose 类：`className="prose prose-lg dark:prose-invert max-w-none"` |
| 页面 padding | `py-20` → `py-16`，标题与内容间距收紧 |
| 阅读进度 | P1 不做，P2 可考虑加顶部细线进度条 |

**文件改动：** `src/app/blog/[slug]/page.tsx`、`src/components/article-renderer.tsx`

### 4.4 标签页

#### 4.4.1 标签聚合页（`/blog/tags`）

**当前状态：** 标签云（链接 + 文章数），无视觉层次。

**提案：**

| 区域 | 改动 |
|------|------|
| 页面标题 | 同首页 Hero 风格 |
| 标签网格 | 从 `flex flex-wrap gap-3` 改为 `grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 gap-3`，视觉更整齐 |
| 标签卡片 | 每个标签加 `bg-[--bg-surface] shadow-sm rounded-xl p-4`，hover `shadow-md` |
| 标签名 | 文字色 `--accent`，字号加大 |
| 文章数 | 单独一行 `--text-muted` 小字 |
| 激活状态 | 当前标签背景 `--accent`，文字白 |

**文件改动：** `src/app/blog/tags/page.tsx`

#### 4.4.2 标签详情页（`/blog/tags/[tag]`）

**当前状态：** 与博客列表页几乎一致，只是多了标签标题。

**提案：**
- 与博客列表页保持一致，但顶部标题区改为：`标签：{name} · {count} 篇`
- 标题左侧加蓝色竖条（与首页一致）
- 标签胶囊显示在标题旁（可点击返回 `/blog/tags`）

**文件改动：** `src/app/blog/tags/[tag]/page.tsx`

## 六、风险与约束

### 6.1 技术风险

| 风险 | 说明 | 缓解 |
|------|------|------|
| `@tailwindcss/typography` 未安装 | 当前未使用 prose 类 | 提案明确需 `npm install @tailwindcss/typography`，修改 tailwind.config |
| Tailwind v4 配置方式 | v4 用 CSS `@theme` 而非 JS config | globals.css 中用 `@theme` 块定义 custom colors |
| 暗色模式覆盖 | 需确保所有新色值都有 dark: 变体 | 配色系统表格中已覆盖 |

### 6.2 设计风险

| 风险 | 说明 | 缓解 |
|------|------|------|
| 蓝色过于突出 | 高饱和蓝可能影响阅读专注度 | 使用 `#2563eb`（sRGB 适中），不追求高饱和 |
| 暗色模式氛围 | 深蓝黑底是否与内容博客气质匹配 | 暗色用 `#0c1222` 而非纯黑，保持深度感，蓝作为强调而非主背景 |
| 改动范围蔓延 | 5 个页面 + 3 个组件，容易超范围 | 严格执行 P1 设计边界，实现由 builder-codex 按 P2 执行 |

### 6.3 约束

- 不修改内容模型、路由结构、认证逻辑
- 不添加新依赖（除 `@tailwindcss/typography`）
- 不改变交互行为（跳转逻辑、状态管理）
- 不做响应式重新设计（保持现有断点逻辑）
- 暗色模式必须完整覆盖，不能半套

## 七、推荐分批上线（Rollout Grouping）

分 3 批上线，每批独立可验收：

### 第一批（视觉核心）— 影响最小，收益最大
1. **globals.css 配色系统**（蓝白 CSS 变量 + dark mode 完整覆盖）
2. **PostCard**（阴影 hover、标签蓝白化）
3. **SiteHeader**（Nav 链接蓝白化）

→ 上线后整体色调立刻转变，用户感知明显，无需改页面结构。

### 第二批（页面结构调整）
1. **首页**（Hero 竖条、CTA 按钮、aside 阴影）
2. **博客列表页**（标题区、标签胶囊）
3. **标签页（/blog/tags）**（网格卡片化）

→ 页面节奏感建立，形成统一的首页-列表页设计语言。

### 第三批（阅读体验精细化）
1. **文章详情页**（宽度收窄、prose 排版、标签位置）
2. **标签详情页**（与博客列表风格统一）
3. **关于页**（分隔线、两栏样式）

→ 最后一批，阅读体验最终收口。

## 八、提案摘要（供 Planner 消费）

```
页面改幅排名（高→低）：首页 > 博客列表 ≈ 标签页 > 文章详情 ≈ 关于页

关键设计决策：
1. 蓝白色系统一全局 CSS 变量，dark mode 完整覆盖
2. 卡片从 border-hover → shadow-hover，减少视觉噪音
3. Hero/标题区统一蓝色竖条锚点，建立视觉识别
4. 标签从 border 小胶囊 → accent-subtle 背景胶囊
5. 文章阅读宽度收窄至 max-w-2xl，引入 typography prose

必须安装：@tailwindcss/typography
建议安装：lucide-react（图标）
涉及文件：12 个（3 新增/改，9 组件/页面）
批次：3 批，批批可独立验收
风险：低（配色方案保守，不改变任何交互逻辑）
```
