## 三、技术实现摘要

### 核心变更

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `src/components/article-renderer.tsx` | 新增（`cf5766b`） | 共用文章渲染组件，供 `/blog/[slug]` 和 `/dashboard/preview/[slug]` 复用 |
| `src/app/blog/[slug]/page.tsx` | 修改（`cf5766b`） | 重构为使用 `<ArticleRenderer post={post} />`，移除内联渲染代码 |
| `src/app/dashboard/preview/[slug]/page.tsx` | 新增（`cf5766b`） | 预览页路由，Server Component，含 auth + 白名单守卫 |
| `src/app/dashboard/page.tsx` | 修改（`cf5766b` + `8d2eff9`） | 草稿行加预览链接 + 移除冗余 guard |

### 关键设计决策

1. **方案 B（独立预览路由）：** 预览页放在 `/dashboard/preview/[slug]`，与 dashboard auth 体系一致，不污染公开博客 SEO
2. **ArticleRenderer 组件抽取：** 将文章主体渲染逻辑抽取为独立组件，两页复用，保证预览与正式页完全一致
3. **Auth 守卫策略：** 未登录/未授权时使用 `notFound()` 而非 redirect，避免泄露受保护路由存在，优于方案设计原始描述
4. **草稿横幅提示：** 预览页顶部添加 `📝 这是草稿预览，内容尚未公开发布` 横幅，区分草稿与正式内容
5. **PostRow guard 简化：** `status === "draft"` 分支中 `post` 必然存在，移除冗余 `post &&` 防护
