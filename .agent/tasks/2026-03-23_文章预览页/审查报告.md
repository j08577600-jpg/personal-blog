### 🔍 风险分析

#### 1. `generateMetadata` 对未发布文章返回完整标题 — **非阻塞**

**描述**：`generateMetadata` 调用 `getPostBySlug(slug)`，对任意 slug（无论 published 状态）均返回 `${post.title}（预览）`。Next.js App Router 中 metadata 路由与页面渲染在同一个受保护的 route segment 中执行，`notFound()` 对未认证用户同样生效，因此实际内容不会泄露。

**结论**：auth 保护有效，风险低。可接受。设计上可考虑将 `generateMetadata` 中的 post 加载合并到主 handler 避免重复调用，但不影响安全性。

#### 2. `post.published` 条件判断正确性 — **通过**

预览页条件 `!post || post.published` → `notFound()`。这意味着：
- 已发布文章（`published: true`）访问预览页 → 404 ✅
- 草稿（`published: false`）→ 正常预览 ✅
- 不存在 → 404 ✅

符合方案设计决策：已发布文章有正式页面，无需预览路由。

#### 3. MDX 渲染安全 — **通过**

`ArticleRenderer` 使用 `next-mdx-remote/rsc` 渲染 `post.content`。`post.content` 来自文件系统（`content/posts/*.mdx`），属于可信内容源，无 XSS 风险。

#### 4. 路由不存在性泄露 — **通过**

未登录/未授权时使用 `notFound()` 而非 redirect，符合安全最佳实践，不泄露受保护路由存在。
