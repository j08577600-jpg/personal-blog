## 实施步骤

### Step 1: 抽取 ArticleRenderer 组件

- **文件**：`src/components/article-renderer.tsx`（新建）
- **内容**：将 `src/app/blog/[slug]/page.tsx` 中的文章主体渲染部分抽取为独立组件 `ArticleRenderer`
  - 接收 `post: Post` 作为 prop
  - 包含：标题区、日期/阅读时间、标签、MDXRemote 渲染
  - **不包含**：「返回博客」链接（各页面自行决定是否加）
- **命令**：`npm run lint`
- **预期结果**：新组件编译通过，无 lint 错误

### Step 2: 重构 `/blog/[slug]` 使用 ArticleRenderer

- **文件**：`src/app/blog/[slug]/page.tsx`（修改）
- **变更**：移除内联渲染逻辑，改为导入并使用 `<ArticleRenderer post={post} />`
- **命令**：`npm run lint && npm run build`
- **预期结果**：公开文章页行为不变，构建通过

### Step 3: 创建预览页路由

- **文件**：`src/app/dashboard/preview/[slug]/page.tsx`（新建）
- **内容**：
  1. Server Component，`async function`
  2. `getServerSession(authOptions)` 校验登录态，未登录 `redirect("/login")`
  3. 校验 whitelisted，未授权 `redirect("/unauthorized")`
  4. `getPostBySlug(slug)` 获取文章，若不存在或 published 为 true，调用 `notFound()`
  5. 顶部「返回控制台 →」和「← 返回博客」链接
  6. 使用 `<ArticleRenderer post={post} />` 渲染内容
  7. `generateMetadata`：返回 `title: post.title + "（预览）"`
- **命令**：`npm run lint && npm run build`
- **预期结果**：预览页构建成功，行为符合验收标准

### Step 4: 更新 dashboard PostRow 为草稿添加预览链接

- **文件**：`src/app/dashboard/page.tsx`（修改）
- **变更**：在 `PostRow` 组件的草稿状态分支，将 `<span>草稿</span>` 替换为 `<Link href={/dashboard/preview/${post.slug}}>预览 →</Link>`
- **命令**：`npm run lint && npm run build`
- **预期结果**：dashboard 中草稿行出现可点击的「预览 →」链接

## 决策日志

- Decision：预览页放在 `/dashboard/preview/[slug]` 而非 `/blog/preview/[slug]`
  原因：预览是作者行为，与 dashboard 同属受保护区域；避免污染公开博客路由；语义正确
  日期/作者：2026-03-23 / planner-codex

- Decision：草稿文章（published=false）即使 slug 匹配也视为不存在，不允许预览
  原因：已发布文章有正式页面，无需预览；避免路由歧义
  日期/作者：2026-03-23 / planner-codex

- Decision：预览页 auth 失败时使用 `notFound()` 而非 `redirect("/login")` / `redirect("/unauthorized")`
  原因：避免泄露受保护路由的存在；`notFound()` 是标准 Next.js 行为，不会给未授权用户任何提示
  日期/作者：2026-03-23 / builder-codex
