### 非 Blocking 问题

1. **ReadingItemRenderer 和 NotePage 直接使用 `MDXRemote` 的样式覆盖方式不一致**
   - `reading-item-renderer.tsx` 在组件内部包含 prose 样式类
   - `notes/[slug]/page.tsx` 在页面层包含 prose 样式类，组件只是 `<MDXRemote>`
   - 两者效果相同，但样式封装位置不一致

2. **lib 文件缺少 Zod schema 验证（与 `posts.ts` 风格不一致）**
   - `posts.ts` 使用 Zod 做运行时 frontmatter 验证
   - `reading-list.ts` 和 `notes.ts` 仅做 TypeScript 类型检查
   - V1 阶段可接受（gray-matter 返回 `unknown` 类型），但长期建议对齐

3. **`reading-list.ts` 文件末尾的 `TagEntry` 类型和 `getAllReadingItemTags` 导出了但未使用**
   - 死代码，无实际影响

4. **导航缺失 `/reading-list` 和 `/notes` 入口**
   - `src/lib/site.ts` 的 `navigation` 数组只有 `/blog`, `/projects`, `/blog/tags`, `/about`
   - 用户无法通过顶部导航发现新页面（需通过首页侧边栏或 sitemap 访问）
   - 不影响功能（sitemap 已覆盖），但影响可发现性
