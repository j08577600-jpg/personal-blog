## 实施步骤

### Step 1: 修改 auth.ts，加入白名单逻辑

**文件：** `src/lib/auth.ts`

**改动：**

在 `callbacks.jwt` 中加入白名单判断：

```ts
callbacks: {
  async jwt({ token, user }) {
    // 首次登录时，user.id 即 GitHub 用户 ID
    if (user?.id) {
      token.sub = user.id as string;
    }

    // 白名单判断（仅在环境变量已配置时生效）
    const allowed = process.env.ALLOWED_GITHUB_USERS;
    if (allowed !== undefined && allowed !== "") {
      const allowedIds = allowed.split(",").map((id) => id.trim());
      if (!allowedIds.includes(token.sub as string)) {
        // 不在白名单，返回 null user
        return { ...token, user: null };
      }
    }

    return token;
  },
  async session({ session, token }) {
    if (session.user && token.sub) {
      session.user.id = token.sub as string;
      // 传递白名单状态（null user 时不设置）
      session.user.whitelisted = token.user !== null;
    }
    return session;
  },
},
```

**注意：** 需要在 `session` 回调中同步设置 `whitelisted` 状态。

**预期结果：** auth.ts 通过 TypeScript 编译，lint 无报错。

### Step 3: 更新 dashboard，未授权时重定向

**文件：** `src/app/dashboard/page.tsx`

**改动：**

在 session 检查后，增加白名单检查：

```ts
export default async function DashboardPage() {
  const session = (await getServerSession(authOptions)) as any;

  if (!session?.user) {
    redirect("/login");
  }

  // 新增：白名单检查
  if (session.user?.whitelisted === false) {
    redirect("/unauthorized");
  }

  // ... 原有内容
}
```

**预期结果：** 未授权用户访问 /dashboard 时跳转到 /unauthorized。
