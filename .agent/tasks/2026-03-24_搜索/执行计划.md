- Decision: 采用 Fuse.js 而非服务端搜索
  原因：V1 文件驱动原则，无数据库依赖，部署简单
  日期/作者：2026-03-24 / planner-codex
- Decision: 在 prebuild 和 build 脚本中都调用索引生成脚本
  原因：prebuild 在 `next build` 之前触发；build 脚本内部又串接了一次确保覆盖率
  日期/作者：2026-03-24 / builder-codex
- Decision: Header 中搜索入口仅使用图标（sr-only 文字）
  原因：Header 空间有限，图标搜索入口是常见模式

## 实施步骤

### Step 1: 创建搜索索引生成脚本

- **文件**：新建 `scripts/build-search-index.mjs`
- **命令**：`node scripts/build-search-index.mjs`
- **预期结果**：在 `public/` 目录生成 `search-index.json`，内容为数组，每个元素包含 `{ slug, title, excerpt, tags, date }`
- **说明**：
  - 复用 `src/lib/posts.ts` 的 `getPosts()` 逻辑（直接读取文件，不 import Next.js 代码）
  - 输出 JSON 格式化可读，带 minify 选项供生产使用
  - 脚本独立运行，不依赖 Next.js 环境变量

### Step 2: 集成到构建流程

- **文件**：修改 `package.json`
- **修改**：在 `scripts.build` 前添加索引生成步骤，或新增 `prebuild` 脚本
- **预期结果**：`npm run build` 自动触发索引生成，`public/search-index.json` 存在
- **备选**：如 prebuild 有问题，改为 `postbuild` 脚本或显式在 build 命令中串接

### Step 3: 创建搜索页面

- **文件**：新建 `src/app/search/page.tsx`
- **说明**：
  - Server Component，先读取 `public/search-index.json`
  - 元数据：title 为"搜索"，description 为"搜索博客文章"
  - 布局与 `/blog` 页面风格一致
  - 主区域放 Client Component `<SearchBox />`

### Step 4: 实现搜索 Client Component

- **文件**：新建 `src/components/search-box.tsx`（`"use client"`）
- **依赖**：安装 `fuse.js`（`npm install fuse.js`，记录到 `package.json`）
- **逻辑**：
  - `useState` 保存 query、results、loading 状态
  - `useEffect` 在组件挂载时 fetch `/search-index.json`，初始化 Fuse 实例
  - `useCallback` + debounce（300ms）响应输入变化
  - Fuse 配置：`keys: ['title', 'excerpt', 'tags']`，`threshold: 0.3`，`includeScore: true`
  - 无 query 时显示"请输入关键词"提示
  - 无结果时显示"未找到相关文章"
- **UI**：
  - 搜索输入框居中显示，带搜索图标
  - 结果以 `<PostCard>` 风格列表展示
  - 每个结果显示标题、摘要（截断 100 字）、标签、日期

### Step 5: 添加搜索入口

- **文件**：`src/components/site-header.tsx`
- **修改**：在导航栏添加"搜索"链接（图标 + 文字）
- **预期结果**：Header 中可见搜索入口，指向 `/search`
