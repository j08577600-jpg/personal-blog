## 目标

本功能完成后：
- `public/sitemap.xml` 包含博客所有已发布文章页和静态页路由，构建时自动生成
- `public/feed.xml` 是有效的 RSS 2.0 Feed，包含所有已发布文章元数据，构建时自动生成
- `package.json` 的 `postbuild` 脚本依次执行 sitemap 生成和 feed 生成

## 上下文

- 方案设计：[方案设计.md](./方案设计.md)
- 内容源：`content/posts/*.mdx`，通过 `src/lib/posts.ts` 的 `getPosts()` 读取
- 公开路由：`/`、`/about`、`/blog`、`/blog/[slug]`
- 站点 URL：环境变量 `NEXT_PUBLIC_SITE_URL`（如未设置，fallback 为 `http://localhost:3000`）
- V1 约束：不引入数据库，sitemap/feed 均为静态文件

## 决策日志

- Decision：采用 next-sitemap + 独立脚本生成 Feed
  原因：静态文件性能最优；next-sitemap 成熟；复用已有 getPosts() 零新增 API 成本
  日期/作者：2026-03-23 / planner-codex
- Decision：sitemap config 直接读取文件系统而非复用 `posts.ts` 的 `getPosts()`
  原因：next-sitemap.config.ts 由 Next.js TypeScript 检查器类型验证，引入 posts.ts 的 Zod 依赖链可能导致类型问题。sitemap 的过滤逻辑（published + 必需字段）与 posts.ts 一致，直接用 gray-matter 解析 MDX 更简洁。
  日期/作者：2026-03-23 / builder-codex
- Decision：sitemap 过滤掉 `published: false` 和缺少必需字段（title/date/slug/excerpt）的文章
  原因：只有通过 Zod 验证的合法文章才会被博客路由返回（getPostBySlug）；sitemap 不应包含会 404 的 URL。
  日期/作者：2026-03-23 / builder-codex
- Decision：`Changefreq` 类型在 next-sitemap 包中未导出，在配置文件中定义本地类型别名
  原因：next-sitemap 的 interface.d.ts 中定义了 `Changefreq = 'always'|'hourly'|...` 但未从主模块导出。本地定义避免类型错误。
  日期/作者：2026-03-23 / builder-codex

## 意外与发现

- 发现：`.next/lock` 文件导致 `npm run build` 报 "Another next build process is already running"。删除该文件后恢复。

### Step 2：创建 next-sitemap.config.ts

在项目根目录创建 `next-sitemap.config.ts`：

```typescript
import type { NextConfig } from "next";

const siteUrl = process.env.NEXT_PUBLIC_SITE_URL || "http://localhost:3000";

const config: NextConfig = {
  siteUrl,
  generateRobotsSitemap: false, // 暂不做 robots.txt，V2 再加
  changefreq: "weekly",
  priority: 0.7,
  sitemapSize: 5000,
  exclude: [],  // 暂无需排除的路径
  robots: [],   // 同上
  outDir: "./public",
  // 动态路由：Next.js App Router 格式
  // getServerSideProps 场景用 source 模式
  // 本项目为静态导出或 SSG，next-sitemap 支持基于实际路由的 glob
};

export default config;
```

- 文件：`next-sitemap.config.ts`（新建）
- 命令：无
- 预期结果：文件存在，TypeScript 编译无错误

> **注意**：Next.js App Router + SSG 场景下，`next-sitemap` 需要在 `postbuild` 运行时知道实际路由。如果 `getPosts()` 数量不大（<5000），next-sitemap 可通过 `transform` 回调动态注入 slug 列表。Builder 需检查 next-sitemap 版本对 App Router 的支持方式，必要时改用 `source` 配置或自定义 glob。**如有兼容问题，在决策日志中记录并回退到手动生成 sitemap**（在 `scripts/generate-feed.ts` 中一并生成 sitemap）。

### Step 3：创建 scripts/generate-feed.ts

在项目根目录创建 `scripts/` 文件夹，新建 `generate-feed.ts`：

```typescript
#!/usr/bin/env npx tsx
/**
 * generate-feed.ts
 * 在 postbuild 时运行，生成 public/feed.xml
 */
import fs from "fs";
import path from "path";
// 复用已有的 getPosts 逻辑，注意此脚本在 Node 环境下运行
// 需要处理路径：process.cwd() 在 scripts/ 时会指向项目根目录
import { getPosts } from "../src/lib/posts";

const SITE_URL =
  process.env.NEXT_PUBLIC_SITE_URL || "http://localhost:3000";

function escapeXml(str: string): string {
  return str
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&apos;");
}

function toRfc822(dateStr: string): string {
  // dateStr 格式：yyyy-MM-dd
  return new Date(dateStr + "T00:00:00.000Z").toUTCString();
}

function generateFeed(): void {
  const posts = getPosts().filter((p) => p.published);

  const items = posts
    .map((post) => {
      const url = `${SITE_URL}/blog/${post.slug}`;
      return `  <item>
    <title>${escapeXml(post.title)}</title>
    <link>${url}</link>
    <guid isPermaLink="true">${url}</guid>
    <pubDate>${toRfc822(post.date)}</pubDate>
    <description>${escapeXml(post.excerpt)}</description>
    ${post.tags.map((t) => `<category>${escapeXml(t)}</category>`).join("\n    ")}
  </item>`;
    })
    .join("\n");

  const xml = `<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0" xmlns:atom="http://www.w3.org/2005/Atom">
  <channel>
    <title>${escapeXml("个人博客")}</title>
    <link>${SITE_URL}</link>
    <description>${escapeXml("最新文章更新")}</description>
    <language>zh-CN</language>
    <lastBuildDate>${new Date().toUTCString()}</lastBuildDate>
    <atom:link href="${SITE_URL}/feed.xml" rel="self" type="application/rss+xml"/>
${items}
  </channel>
</rss>`;

  const outPath = path.join(process.cwd(), "public", "feed.xml");
  fs.mkdirSync(path.dirname(outPath), { recursive: true });
  fs.writeFileSync(outPath, xml, "utf8");
  console.log(`✅ feed.xml generated: ${posts.length} posts → ${outPath}`);
}

generateFeed();
```

> **说明**：此脚本作为独立 Node 脚本运行，不经过 Next.js bundler。使用 `npx tsx` 执行。如 `tsx` 未安装，`builder-codex` 需先 `npm install tsx --save-dev`。

- 文件：`scripts/generate-feed.ts`（新建）
- 命令：`npx tsx scripts/generate-feed.ts`
- 预期结果：`public/feed.xml` 生成，内容包含 `<item>` 列表
