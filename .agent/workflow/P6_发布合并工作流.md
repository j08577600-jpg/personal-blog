# P6 发布 / 合并工作流

> 本文档定义 P6 阶段的标准工作流。编排者负责执行，builder-codex 配合执行发布操作。

---

## 触发条件

P5 测试验收阶段完成，且：
- 所有测试项通过
- 审查报告中无阻塞项

---

## 工作流步骤

### Step 1：收集阶段文档

收集本任务所有阶段文档：

```
.agent/tasks/[yyyy-MM-dd_功能名]/
├── yyyy-MM-dd_[功能]_方案设计.md      ← P1
├── yyyy-MM-dd_[功能]_执行计划.md      ← P2（含完成摘要）
├── yyyy-MM-dd_[功能]_审查报告.md      ← P4
└── yyyy-MM-dd_[功能]_测试报告.md      ← P5
```

**检查清单：**
- [ ] P1 方案设计文档存在
- [ ] P2 执行计划文档存在（含完成摘要）
- [ ] P4 审查报告文档存在（含综合结论）
- [ ] P5 测试报告文档存在（含最终结论）

> 如果某阶段无对应文档（P2 可能在执行过程中直接写入代码变更），跳过即可。

---

### Step 2：生成最终报告

#### 2.1 编写最终报告

编排者根据收集到的文档，生成最终报告。

**文件名格式：** `yyyy-MM-dd_[功能]_最终报告.pdf`

**存放位置：** `.agent/tasks/[yyyy-MM-dd_功能名]/`

**报告结构：**

```
# [功能名] - 最终交付报告

> 日期：yyyy-MM-dd
> 任务：[功能名]
> 状态：✅ 发布完成

## 一、项目概述
- 任务名称
- 目标
- 框架
- 协作规范

## 二、阶段交付物
- P1 / P2 / P3 / P4 / P5 各阶段的负责人、文档、结论

## 三、技术实现摘要
- 核心逻辑（关键代码片段）
- 环境变量（如有）

## 四、验收标准达成情况
- 逐项打勾

## 五、已知项与后续
- 遗留问题（如有）
- V2 规划（如有）

## 六、交付物清单
- 文档清单
- 代码变更清单

---
_报告生成时间：yyyy-MM-dd HH:MM UTC_
_编排者：[编排者名]_
```

#### 2.2 导出 PDF

只输出 PDF，不保留中间文件。

```bash
google-chrome --headless --disable-gpu --no-sandbox \
  --print-to-pdf="[任务目录]/yyyy-MM-dd_[功能]_最终报告.pdf" \
  --print-to-pdf-no-header "file://[任务目录]/yyyy-MM-dd_[功能]_最终报告.html"
```

---

### Step 3：Git 合并

#### 3.1 确认分支状态

```bash
cd /home/cjl/projects/personal-blog
git status
git log --oneline -5
```

#### 3.2 合并到 main

```bash
# 切到 main，拉取最新
git checkout main && git pull

# 合并功能分支
git merge [分支名]

# 如有冲突，记录冲突文件，通知 builder-codex 解决
```

#### 3.3 推送

```bash
git push origin main
```

**检查清单：**
- [ ] 合并无冲突（或冲突已解决）
- [ ] main 分支已推送
- [ ] 功能分支可选择保留或删除

---

### Step 4：发布

#### 4.1 构建验证

```bash
npm run build
npm run lint
```

#### 4.2 部署到生产

```bash
# PM2 重启（如已配置）
pm2 restart personal-blog

# 或其他部署方式
```

> 具体部署方式见项目 `deploy/` 目录。

**检查清单：**
- [ ] build 通过
- [ ] lint 通过
- [ ] 服务已重启
- [ ] 访问首页正常

---

### Step 5：收尾

- [ ] 最终报告 PDF 已生成并归档
- [ ] 更新 `DEVELOPMENT_PLAN.md`，将任务从"进行中"移到"已完成"
- [ ] 如有遗留问题，写入项目 issue 或记录到 `DEVELOPMENT_PLAN.md` 的"后续"区
- [ ] 向 jay 汇报完成状态

## 自动化脚本

```bash
#!/usr/bin/env bash
# P6 自动化脚本（部分）
# 用法：bash .agent/scripts/p6-publish.sh [分支名] [功能名]

BRANCH="${1}"
FEATURE="${2}"
TASK_DIR=".agent/tasks/${FEATURE}"

if [ -z "$BRANCH" ] || [ -z "$FEATURE" ]; then
  echo "用法：bash p6-publish.sh [分支名] [功能名]"
  exit 1
fi

set -e

echo "=== P6: 收集阶段文档 ==="
ls "${TASK_DIR}/"

echo "=== P6: Git 合并 ==="
git checkout main && git pull
git merge "${BRANCH}"
git push origin main

echo "=== P6: 构建验证 ==="
npm run build
npm run lint

echo "=== P6 完成 ==="
```
